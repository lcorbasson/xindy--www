	Hello,

just a few words about the 8-bit problems mentioned in the previous
Mail by Ralf Höller.

xindy is completely 8-bit clean. It is not made to work with TeX/LaTeX
alone. The underlying document alphabet it operates on is ISO Latin 1.
But even this does't matter since it only sees an alphabet of 256
characters. The only restriction is that the encoding scheme of the
raw-index and the index style should be equivalent. Not a problem if
some scheme such as ISO Latin or ASCII is used for all raw- and
style-files.

If there are problems with 8-bit in TeX/LaTeX, this is not a problem
of the core xindy system.
